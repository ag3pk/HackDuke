1.1.1 (07 Feb 2013):

New Features

  • Experimental support for devices using the x86 and MIPS architectures.

Bug Fixes

  • Blacklist some devices/chipsets from using OpenSL ES. They seem to always error out, claiming that the microphone device is already open and can't be opened a second time. These devices will fall back to using JNI AudioTrack audio.
  • Fix an issue where a terminated call might not be cleaned up properly, resulting in a RuntimeException thrown from the native code.
  • Fix a memory corruption issue that could cause crashes in the native library.
  • Fix a problem where, after a network disconnect/reconnect, the SDK is unable to make or receive new calls.
  • Add missing DeviceListener logging in the BasicPhone sample app.

1.1.0 (19 Sep 2012)

New Features

  • Increase keepalive interval to bring down device wakeups and reduce idle network usage.
  • Use the Speex wideband codec rather than narrowband, which should result in better voice quality at the expense of slightly greater in-call network usage.
  • Signalling will now recover after switching between WiFi and mobile data (or between different WiFi networks), so if the other party hangs up, the mobile device will receive the notification.

Bug Fixes

  • Fix a possible crash when initiating a new call.
  • Fixed a DNS server error when connected to networks that provide IPv6 DNS server addresses.
  • Fixed a possible reference cycle that could lead to a Device object memory leak.
  • Fixed an issue where a Device could receive duplicate incoming call notifications after a network disconnect and reconnect.
  • Fixed a case where a connection disconnect() immediately after a connect() could sometimes still allow the call to be placed. In some other cases this would result in the connection listener's onDisconnected() method being called with an error reported. This is now just reported as a non-error disconnect.
